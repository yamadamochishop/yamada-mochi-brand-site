# 山田もち店 CMS比較と導入方針

山田もち店のサイトは、まずは `data/products.ts` や `data/news.ts` のようなデータファイルで一元管理できる構成にしています。

理由は、最初の公開までを早く・安全に進めるためです。商品名、価格、原材料、説明文、画像パスを1か所で管理し、ホームページ全体へ反映できます。

将来的に「管理画面から商品やお知らせを編集したい」となった段階で、ヘッドレスCMSを追加するのがおすすめです。

## 比較

| CMS | 向いている使い方 | 良い点 | 注意点 |
| --- | --- | --- | --- |
| Sanity | 商品情報・お知らせ・写真ギャラリーを管理画面で更新 | 柔軟。Next.jsとの相性が良い。日本語入力も問題なく、画像管理もしやすい | 初期設定は少し開発作業が必要 |
| Contentful | 大きめの企業サイト、複数人運用 | 管理画面が整っていて安定感がある | 無料枠や項目設計に制限を感じる場合がある |
| Decap CMS | GitHub上のMarkdown/JSONを管理画面で編集 | データがGitHubに残る。静的サイトと相性が良い | ログイン設定や画像管理がややクセあり |

## 現時点のおすすめ

現時点では、まず `data/` フォルダで公開し、次の段階で **Sanity** を導入する方針がおすすめです。

理由:

- 商品、ギフト、お知らせ、写真ギャラリーを管理画面化しやすい
- Next.js / Vercelとの相性が良い
- 将来的に英語・中国語・韓国語対応へ広げやすい
- 画像の差し替えや商品追加がしやすい

## 導入タイミング

おすすめの順番:

1. Next.js版をVercelで公開
2. Google Search Console / Analyticsを設定
3. 商品画像や文章を整える
4. 更新頻度が増えてきたらSanityを導入

最初からCMSを入れることもできますが、公開前の作業が増えます。まずブランドサイトを公開し、運用しながらCMS化する方が安全です。

## CMS化したいデータ

- 商品情報
- ギフトセット
- お知らせ
- お客様の声
- FAQ
- ギャラリー画像
- レシピ・焼き方記事

現在の設計では、これらを後からCMSへ移行しやすいよう、データをページ内に直接書かず、`data/` にまとめています。
